zds_get_attr_string

int zds_get_attr_string(zds_htile hControl,char *szAttribName, char *szAttribInfo, long lInfoMaxLength);

Retrieves the DCL value of the specified attribute, using the tile handle that is passed into the callback function.

Similar to zds_get_attr() but used inside callback functions, where the tile handle hControl is passed as the zds_htile field of the callback packet. The szAttribName argument, which is case sensitive, specifies the name of the attribute. If zds_get_attr_string() succeeds, it returns RTNORM and sets szAttribInfo to the attribute's DCL value; lInfoMaxLength specifies the length that the function has allocated to szAttribInfo. The lInfoMaxLength argument should include room for the string terminator (\0 or EOS). If zds_get_attr_string() can't find the specified attribute, it returns RTERROR.

This function is a bit more efficient than zds_get_attr(), because the tile handle indicates which tile the user selected and eliminates the need for another string argument.

The value that is returned is the attribute's initial value as specified in its DCL description. It does not reflect changes to the state of the tile that come through user input or zds_set_tile() calls.

The function returns RTNORM if successful; otherwise; it returns an error code.

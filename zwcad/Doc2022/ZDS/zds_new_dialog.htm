zds_new_dialog

int zds_new_dialog(char *szDialogName,int nDialogFileID,void *pfnDef_Callback,zds_hdlg *hDialog);

Begins handling a new dialog box, displays it, and optionally specifies a default action.

The szDialogName argument is a string that specifies the dialog box, and nDialogFileID identifies the DCL file (you must have obtained its value from the zds_load_dialog() call).

Your application must call zds_new_dialog() before it calls zds_start_dialog(). All dialog box initialization, such as setting tile values, creating images or lists for list boxes, and associating actions with specific tiles (with zds_action_tile()) must take place after the zds_new_dialog() call and before the zds_start_dialog() call.

The default action is evaluated when the user selects an active tile that doesn't have an action or callback explicitly assigned to it by zds_action_tile() or in DCL.

The pfnDef_Callback argument specifies the optional default callback function. If you don't use this option, pfnDef_Callback must be NULLCB. If zds_new_dialog() succeeds, it returns RTNORM and sets hDialog to a valid dialog box handle; otherwise, it returns RTERROR and sets hDialog to NULL. Many subsequent calls to programmable dialog box (PDB) functions require you to specify the hDialog value that the call to zds_new_dialog() returns.

Note Always check the status that zds_new_dialog() returns. Calling zds_start_dialog() when the zds_new_dialog() call has failed can have unpredictable results.

The function returns RTNORM if successful; otherwise, it returns an error code.

zds_slide_image

int zds_slide_image(int nULCornerX, int nULCornerY, int nWidthLess1, int nHeightLess1, char *szSnapshotName);

Draws an ZWCAD slide in the currently active image.

The slide can be a slide (.sld) file or a slide in a slide library (.slb) file. The szSnapshotName argument specifies the type of file in the same way as the VSLIDE command or a menu file, as follows:

libname(sldname)

The first (upper-left) corner of the slide -- its insertion point -- is located at (nULCornerX, nULCornerY), and the second (lower-right) corner is located at the relative distance (nWidthLess1, nHeightLess1) from the first corner (nWidthLess1 and nHeightLess1 must be positive values). The origin (0,0) is the upper-left corner of the image. You obtain the coordinates of the lower-right corner by calling the dimension function zds_dimensions_tile()).

Slides act like transparencies; they usually don't display all the pixels in the region they cover. Make sure that the pixels that the slide does display contrast with the background of the image. Vectors in slides are often drawn in white (color number 7), which is the default background color of an image. If your image tile is blank when you first display a slide, try changing its color attribute to graphics_background (usually equivalent to color number 0). (You can also change the background of the image by preceding the zds_slide_image() call with an zds_fill_image() call.)

Because slides act like transparencies, you can build an image by superimposing multiple slides. Displaying slides can be time-consuming. If you use them, keep them simple.

Solid filled areas do not display filled unless the slide was created from an image generated by the SHADE command.

The zds_slide_image function returns RTNORM if successful; otherwise, it returns an error code.

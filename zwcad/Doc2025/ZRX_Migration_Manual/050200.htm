Because some functions of AcGsManager have changed, getGSClassFactory() no longer exists, the AcGsGraphicsKernel class has replaced the AcGsClassFactory class, and GS functions such as createAutoCADModel also need to specify a specific kenel to create. Get kernel method by previous：

AcGsGraphicsKernel *pKernel = pGsManager->getGSClassFactory();

Change to:

AcGsKernelDescriptor descriptor;

descriptor.addRequirement(AcGsKernelDescriptor::k3DDrawing);

AcGsGraphicsKernel *pKernel = AcGsManager::acquireGraphicsKernel(descriptor);

But because AcGsKernelDescriptor class and AcGsManager: : acquireGraphicsKernel method internal implementation has not been completed, so the ZWCAD still use the original method:

AcGsGraphicsKernel* kernel= pGsManager ->getGSClassFactory();

Get and use the kernel, and you cannot create Model, Device, View, and so on with the specified kernel.The internal implementation of GS functions such as createZWCADModel(ZcGsGraphicsKernel &kernel) still follows the original logic, and the incoming kernel parameters are not used.

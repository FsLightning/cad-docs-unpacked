 Custom Entity UI
 

Detailed Description

All database objects with graphical representations are derived from AcDbEntity. If a class is derived from AcDbEntity,it is a visible object,In addition we can define it shape object ,snap points, grip points, and stretch points and so on. It is recommended that the data class and the operation be separated when building a custom entity, the advantage is that if a custom entity is present in the drawing,only need to the data class is provided and loaded to display the custom entity instead of the proxy class. If others want to operate a custom entity you can then provide and load the operation class, and if you want to create a custom drag sequence for your entity, you will implement your own AcEdJig, and we'll show you how to drag your custom entity in the next section.

Input Options for User-Input Functions
Basic Interaction with ZWCAD » Global Functions for Interacting with ZWCAD » Getting User Input » Control of User-Input Function Conditions

In the interaction with the user to obtain data, sometimes need to add some restrictions. Such as prompting the user to enter the radius of the circle, it must be limited to the input data can not be negative. This can be done by calling the acedInitGet () function. The function will control the user input followed by a acedGetXXX () function behind it.

int acedInitGet(int val,const ACHAR *kwl);

Input options set by acedInitGet:

  • 1 RSG_NONULL Disallow null input
  • 2 RSG_NOZERO Disallow zero values
  • 4 RSG_NONEG Disallow negative values
  • 8 RSG_NOLIM Do not check drawing limits, even if LIMCHECK is on
  • 32 RSG_DASH Use dashed lines when drawing rubber-band line or box
  • 64 RSG_2D Ignore Z coordinate of 3D points (acedGetDist() only)
  • 128 RSG_OTHER Allow arbitrary input–whatever the user types
  • 256 RSG_DDISTFIRST Force direct distance input to have precedence over arbitrary input
  • 512 RSG_TRACKUCS Allow UCS tracking for faces
  • 1024 RSG_NOORTHOZ Disables polar/otrack/ortho tracking in the Z direction
  • 2048 RSG_NOOSNAP Disables object snaps
void InteractGlobalFunction::UserInputInitGet()
{
int num = 0;
acedInitGet(RSG_NONULL | RSG_NOZERO | RSG_NONEG,NULL);
int rc=acedGetInt(_T("Input int Number:\n"), &num);
if ( rc== RTNORM)
{
acutPrintf(_T("Input Number:%f\n"),num);
}
}

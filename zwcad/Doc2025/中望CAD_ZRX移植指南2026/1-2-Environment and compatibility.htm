中望CAD 2026 ZRX移植指南
环境要求

总览

中望CAD提供ZRXSDK支持开发者在中望平台上进行二次开发，开发者可以使用ZRXSDK中的接口来实现各种功能。开发者需要先配置开发环境，然后使用ZRXSDK中的接口编写程序，然后编译生成ZRX应用程序，最后在中望CAD中加载运行。一般情况下，中望CAD连续3~4个年度版本内不会修改ZRXSDK接口，使用旧版本SDK编译的ZRX可以新版本上加载，所以在二进制兼容版本内开发者无需修改源码和编译即可进行中望平台年度版本的升级。

ZRXSDK版本(二进制兼容) 开发环境 平台工具集 .NET Framework版本 32位支持 对应ObjectARX版本
ZRXSDK2017~ZRXSDK2020 Microsoft Visual Studio 2010 v140 .NET Framework 4.0 是 ObjectARX2008
ZRXSDK2021~ZRXSDK2024 Microsoft Visual Studio 2017 15.9及以上 v141 .NET Framework 4.7 是 ObjectARX2019
ZRXSDK2025~ Microsoft Visual Studio 2017 15.9及以上 v141 .NET Framework 4.7 否 ObjectARX2024

二进制兼容性

中望2021~中望2024版本的二进制相互兼容，可以基于ZRXSDK2021(22/23/24)进行二次开发，接口和ObjectARX2019兼容。中望2025版本升级了ZRXSDK，接口和ObjectARX2024兼容，需要基于ZRXSDK2025重新编译ZRX文件，然后才能在中望2025上加载使用。中望2026版本的接口和中望2025兼容，中望2025编译的ZRX无需重新编译，可以直接在中望2026上加载使用。

开发环境

推荐使用Microsoft Visual Studio 2017(15.9及以上版本)进行二次开发，Visual Studio平台工具集使用v141，.NET Framework使用4.7。

操作系统

推荐使用Windows 10 (1703)及以上版本，具体见.NET Framework 4.7系统支持。

32位支持

中望2025版本开始不再支持32位。

ZRXSDK目录结构

ZRXSDK中的arxport文件夹模拟了ARXSDK的文件结构，便于开发者从ARX移植到ZRX。当然，您也可以直接包含和引用ZRXSDK的inc和lib目录进行开发。

注意 当前ZRX接口和ObjectARX2024兼容，如果您是从ObjectARX低版本移植到中望2026，那么建议先将接口升级到ObjectARX2024，然后再移植到中望2026。
